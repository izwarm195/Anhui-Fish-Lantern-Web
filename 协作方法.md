### **二、3D开发者的职责**

入口文件只需要这样调用：

```js

const scene = new FishLanternScene(
```

3D开发者需要遵守几个边界：

- 只操作 `#scene-container` 内部的 Canvas。
- 不修改 `.nav-wrapper`、`.info-overlay`、`.site-title` 等UI元素。
- 不在场景模块中写鱼灯简介、历史、制作工艺等文字内容。
- 模型、纹理和字体都放入 `public/`，不要散落在源码目录中。
- 对外只暴露初始化、尺寸调整、销毁，以及必要的交互事件。

如果未来需要替换真实鱼灯模型，3D开发者只需替换 `lanternModel.js`，UI开发者不需要改动任何代码。

### **四、提前约定两个人之间的接口**

两个人独立开发的关键不是“尽量不碰同一个文件”，而是提前约定通信方式。

#### **1. 视觉层接口**

双方先固定以下 DOM 结构：

```html
<div id="scene-container"></div>
<div id="ui-layer"></div>
```

CSS 中明确层级：

```css
#scene-container {
  position: fixed;
  inset: 0;
  z-index: 1;
}

#ui-layer {
  position: fixed;
  inset: 0;
  z-index: 10;
  pointer-events: none;
}

#ui-layer button,
#ui-layer .nav-item,
#ui-layer .info-overlay {
  pointer-events: auto;
}
```

这样3D画布始终在底层，UI始终在上层，两边不会因为定位和层级互相覆盖。

#### **2. 事件接口**

如果UI需要控制3D场景，例如点击“鱼灯简介”时暂停旋转，可以使用自定义事件：

```js
// UI模块触发
window.dispatchEvent(new CustomEvent('lantern:pause'));
```

3D模块监听：

```js
window.addEventListener('lantern:pause', () => {
  this.controls.autoRotate = false;
});
```

恢复播放：

```js
window.dispatchEvent(new CustomEvent('lantern:resume'));
```

3D场景也可以通知UI：

```js
window.dispatchEvent(new CustomEvent('lantern:ready'));
```

UI模块监听：

```js
window.addEventListener('lantern:ready', () => {
  document.body.classList.add('scene-ready');
});
```

事件名称最好统一使用命名空间，例如：

```text
lantern:ready
lantern:pause
lantern:resume
lantern:focus
lantern:model-loaded
lantern:model-error
```

不要直接让UI代码访问：

```js
scene.children[3].rotation.y += 0.1;
```

这种写法会让两个模块强耦合，后续替换模型或调整场景结构时很容易出错。

### **六、开发阶段不要直接使用真实模型也可以**

在真实鱼灯模型尚未完成时，3D开发者可以继续使用当前的立方体占位模型，并保持模型加载接口不变：

```js
export async function loadLanternModel(scene) {
  // 当前阶段使用占位模型
  return createPlaceholderLantern();
}
```

之后替换为真实模型：

```js
export async function loadLanternModel(scene) {
  const loader = new GLTFLoader();
  const gltf = await loader.loadAsync('/models/fish-lantern.glb');
  return gltf.scene;
}
```

只要返回的都是一个 `THREE.Object3D`，场景控制器、灯光和动画逻辑就不需要重新设计。

真实模型建议优先使用 `.glb` 或 `.gltf`。模型文件应统一约定：

- 模型中心位于原点；
- 鱼灯底部位于约定的高度；
- 单位统一；
- 初始朝向统一；
- 模型名称固定；
- 纹理使用相对路径；
- 文件大小控制在合理范围内。

### **七、建议的整合入口**

最终的 `main.js` 可以只负责组装模块：

```js
import { FishLanternScene } from './scene/FishLanternScene.js';
import { Navigation } from './ui/navigation.js';
import { Modal } from './ui/modal.js';
import './styles/global.css';

const scene = new FishLanternScene(
  document.querySelector('#scene-container')
);

const modal = new Modal({
  container: document.querySelector('#info-modal')
});

const navigation = new Navigation({
  container: document.querySelector('#site-navigation'),
  onSelect: (topic) => {
    modal.open(topic);
  }
});

scene.init();
modal.init();
navigation.init();
```

这个文件不应该包含具体的灯光参数、粒子算法、弹窗样式或文案，只负责“创建模块并连接模块”。

### **八、最终联调顺序**

建议按照下面的顺序合并：

1. 先确定页面容器、层级和基础尺寸；
2. 合并3D场景，确认Canvas可以全屏显示；
3. 合并导航和弹窗，确认UI不会阻止Canvas拖拽；
4. 接入自定义事件；
5. 替换真实鱼灯模型；
6. 进行移动端、低性能设备和不同浏览器测试；
7. 最后统一颜色、字体、动画速度和响应式细节。

需要特别测试一个问题：UI透明区域不能覆盖整个页面的鼠标事件，否则 Three.js 的 `OrbitControls` 无法拖动。可以让UI层使用：

```css
#ui-layer {
  pointer-events: none;
}

#ui-layer .interactive-element {
  pointer-events: auto;
}
```
