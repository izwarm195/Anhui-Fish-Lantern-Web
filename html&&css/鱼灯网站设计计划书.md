## 一、设计理念

### 1.1 核心概念

**"河流叙事"** — 以河流作为视觉线索和叙事主轴，将徽州鱼灯文化植根于其地理语境中。观众像沿河游览一样，顺流而下依次了解鱼灯的历史文化、制作工艺、实地观赏信息。

### 1.2 视觉锚点

- **夜间场景**：鱼灯是夜间巡游的非遗活动，暗黑色调还原真实的观赏体验
- **暖金色流光**：鱼灯标志性色调，页面中唯一的亮色系，在暗夜中格外夺目
- **徽派建筑**：粉墙黛瓦、马头墙、暖黄窗灯，将鱼灯文化扎根于徽州地理
- **河流**：横贯画面的 SVG / 3D 河流，既是视觉线索也是叙事主轴

### 1.3 情感目标

> "一个人点亮一盏鱼灯 → 鱼灯文化繁衍扩散"

从一条鱼灯到三条鱼灯的转变，隐喻鱼灯文化在徽州大地的传承与扩散。让观众在浏览器中完成"了解鱼灯 → 拆解工艺 → 查询实地观赏地点"的完整体验，最终产生"想去现场看看"的冲动。

## 三、技术架构

### 3.1 技术选型

```
┌──────────────────────────────────────────────┐
│                浏览器 (file://)               │

│                 原生 HTML + CSS + JS          │

├──────────────────────┬───────────────────────┤
│   Three.js (ESM)     │   GSAP (UMD)          │
```

## 五、交互细节

### 5.1 开场画面

**视觉构成**：
- 夜幕星空，月光从左上角斜照
- 河流横贯画面下 1/3 处，微微波动
- 两岸徽派建筑剪影，近处窗户透暖黄灯光
- 80 只萤火虫粒子在河面与建筑间飘舞
- 鱼灯在河面上方绕"开始"做椭圆轨迹转动

**鱼灯非匀速转动规律**：
```
速度曲线：
  ▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁████████▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁████
  ↑ 慢速巡游（蓄力） ↑ 扑腾爆发    ↑ 慢速        ↑ 扑腾
  ←── 2.5s ──→   ←0.6s→  ←── 2.5s ──→   ←0.6s→
```
- 大部分时间慢速巡游（蓄力）
- 每 2.5s 一次"扑腾"爆发（速度提升 3 倍），持续 0.6s
- 轨道为椭圆（横 3.8、纵 2.2），叠加上下浮动 ±0.2
- 鱼灯朝向运动方向，带 `lerp` 惯性平滑

**"开始"按钮**：
- 固定在画面正中央，鱼灯围绕它转动
- 双层光环呼吸动画
- hover 放大 + 光晕增强
- 点击后立即触发转场

### 5.2 转场动画

```
时间线（GSAP）:
0s     ─────────────────────────────────────── 2.5s
│                                          │
├─ 0.8s: 鱼灯加速半圈 (power2.in)          │
├─ 1.0s: 鱼灯冲向观者，Z轴 +5 (power3.out)  │
├─ 0.2s: 鱼灯淡出消失                       │
├─ 0.3s: 拖尾淡出                           │
├─ 2.5s: 镜头推进 (Z: 12→6, power2.inOut)  │
│                                          │
2.5s ─────────────────────────────────────── 5.5s
│                                          │
├─ 0.8s: 第一条鱼灯从河面升起 (back.out)     │
├─ 1.0s: 第二条鱼灯升起                      │
├─ 1.2s: 第三条鱼灯升起                      │
├─ 2.5s: 板块标记逐个 fadeIn + 上移          │
│                                          │
5.5s ─────────────────────────────────────── 进入探索
```

### 5.3 探索阶段（预留）

- 页面变为可滚动
- ScrollTrigger 将滚动映射为相机沿河流移动
- 三条鱼灯缓慢自转，各自跟随对应板块
- 每到达一个板块，内容以河流动画引导方式淡入

## 八、当前问题记录

### 8.1 打开空白 / 页面进不去

可能的检查点：
- [ ] 浏览器是否支持 ES Module（Chrome 89+ / Firefox 108+）
- [ ] importmap 是否被浏览器支持（Chrome 89+）
- [ ] 所有 CDN URL 是否能正常访问
- [ ] 控制台是否有 CORS 错误（file:// 协议不能加载某些 CDN）
- [ ] 控制台是否有模块加载错误（404 / MIME type 校验）

### 8.2 后续注意事项

- Three.js 0.160.0 只有 ESM 构建，必须用 importmap 引入
- GSAP 3.12.5 有 UMD 构建，可以用 `<script>` 标签
- `file://` 协议下部分 CDN 可能因 CORS 拒绝加载，如遇此情况可尝试换用 CDN 或开一个本地服务器（`npx serve .`）
